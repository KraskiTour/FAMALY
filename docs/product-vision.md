# Product Vision — ЮгТур

## Описание продукта

ЮгТур — это современный туристический каталог автобусных туров из южных городов России.

Сайт является полноценным продуктовым туристическим порталом:
- каталог с фильтрацией и сортировкой
- SEO-оптимизированные страницы направлений и городов выезда
- подробные карточки туров с маршрутом, ценами и датами
- архитектура, готовая к масштабированию на сотни туров

Это **не лендинг** и не визитка. Это инструмент продаж и SEO-захвата трафика.

---

## Для кого сайт

| Сегмент | Описание |
|---|---|
| Взрослые туристы 30–60 лет | Ищут организованный отдых с комфортом |
| Семейные туристы | Мамы с детьми, пары — хотят безопасный отдых без хлопот |
| Любители активного отдыха | Горы, природа, маршруты выходного дня |
| Пенсионеры | Доступные цены, простота бронирования |

**География:** Краснодар, Ростов-на-Дону, Ставрополь, Новороссийск, Сочи.

---

## Какие задачи решает

1. Быстро показать доступные туры по городу выезда
2. Помочь выбрать тур по направлению, датам, длительности и бюджету
3. Дать полную информацию о туре: маршрут, цена, что включено
4. Привести пользователя к заявке или контакту в мессенджере
5. Генерировать SEO-трафик через страницы городов и направлений

---

## Чем отличается от лендинга

| Лендинг | Каталог ЮгТур |
|---|---|
| Одна страница | Десятки/сотни страниц |
| Один оффер | Каталог с фильтрами |
| Нет SEO-структуры | Полная SEO-архитектура |
| Нельзя масштабировать | Легко добавлять туры и города |
| Быстро устаревает | Актуален при обновлении данных |

## Главные пользовательские сценарии

### Сценарий 1: «Хочу поехать из Краснодара»
Пользователь заходит → видит город выезда → выбирает направление → смотрит карточку тура → оставляет заявку.

### Сценарий 2: «Хочу в горы на выходные»
Пользователь заходит → открывает каталог → фильтрует по длительности «2–3 дня» → выбирает тур → пишет в WhatsApp.

### Сценарий 3: SEO-вход
Пользователь ищет «автобусный тур в Домбай из Ростова» → попадает на страницу направления → переходит к конкретному туру.

### Сценарий 4: «Сравниваю варианты»
Пользователь просматривает каталог → сравнивает цены и даты → выбирает оптимальный вариант.

---

## Бизнес-цели

1. **Лидогенерация** — заявки через форму, WhatsApp, Telegram
2. **SEO-трафик** — органический трафик через страницы городов и направлений
3. **Конверсия в продажу** — удобный путь от каталога до заявки
4. **Доверие** — внешний вид и контент формируют образ надёжного туроператора
5. **Масштабируемость** — лёгкое добавление новых туров без разработчика

---

## Визуальные принципы

- **Чистота** — много воздуха, минимум визуального мусора
- **Премиальность** — выглядит дороже среднего турагентства
- **Контрастность** — чёткая типографика, читаемые цены
- **Фото-ориентированность** — крупные качественные изображения
- **Единая система** — одинаковые отступы, радиусы, тени, состояния

---

## UX-принципы

- Цена, дата и город выезда видны на карточке сразу
- Фильтры простые и понятные, без перегрузки
- CTA-кнопка всегда в зоне видимости
- Мобильная версия такая же сильная, как десктопная
- Навигация логичная: главная → каталог → тур → заявка
- Минимум кликов до целевого действия
