# Инструкция: как добавить реальный тур на сайт KRASKI.TRAVEL

## Быстрый чеклист

1. Открыть `data/mock-tours.ts`
2. Скопировать шаблон тура из комментария в начале файла
3. Вставить в массив `tours[]`
4. Заполнить поля (подробности ниже)
5. Положить фото в `public/images/tours/<slug>/`
6. Сохранить файл
7. Проверить в браузере: `http://localhost:3000/tours/<slug>`

---

## 1. Куда добавлять тур

Файл: **`data/mock-tours.ts`**

Найдите массив `export const tours: Tour[] = [` и вставьте новый объект перед закрывающей скобкой `];`.

Порядок туров в массиве = порядок на сайте. Новые туры лучше добавлять **в начало** массива.

---

## 2. Обязательные поля (★)

| Поле | Что заполнить | Пример |
|------|--------------|--------|
| `id` | Уникальный номер (строка) | `'13'` |
| `slug` | URL-адрес: латиница, дефисы, без пробелов | `'arkhyz-weekend'` |
| `title` | Название тура как на сайте | `'Архыз — горный отдых на выходные'` |
| `shortDescription` | 1-2 предложения для карточки в каталоге | |
| `fullDescription` | 3-5 предложений для страницы тура | |
| `departureCities` | Массив городов выезда (см. ниже) | |
| `destination` | Направление (должно совпадать с `destinations[]`) | `'Архыз'` |
| `region` | Регион | `'Кавказ'` / `'Крым'` / `'Абхазия'` / `'Поволжье'` |
| `durationDays` | Число дней | `2` |
| `priceFrom` | Минимальная цена (число) | `5900` |
| `oldPrice` | Старая цена или `null` | `7500` или `null` |
| `nextDates` | Массив ближайших дат (см. ниже) | |
| `included` | Массив строк: что входит | `['Проезд', 'Проживание']` |
| `excluded` | Массив строк: что не входит | `['Личные расходы']` |
| `itinerary` | Программа по дням (см. ниже) | |
| `gallery` | Массив путей к фото (см. ниже) | |
| `isPublished` | `true` = виден, `false` = скрыт | `true` |

## 3. Желательные поля (☆)

| Поле | Что заполнить | Пример |
|------|--------------|--------|
| `badges` | Бейджи на карточке | `['family', 'mountains']` |
| `transport` | Тип транспорта | `'Комфортабельный автобус'` |
| `meals` | Питание | `'Завтраки включены'` |
| `hotel` | Проживание | `'Отель 3* в Архызе'` |
| `difficulty` | Сложность | `'easy'` / `'medium'` / `'hard'` |
| `seasonMonths` | Месяцы проведения | `[5, 6, 7, 8, 9]` |
| `seoTitle` | Заголовок для Google | `'Тур в Архыз из Краснодара'` |
| `seoDescription` | Описание для Google (до 160 символов) | |

## 4. Необязательные поля (○)

| Поле | Что заполнить |
|------|--------------|
| `highlights` | Ключевые фишки: `['Горные панорамы', 'Подходит детям от 5 лет']` |
| `maxGroupSize` | Максимум человек в группе: `45` |
| `minGroupSize` | Минимум для подтверждения: `10` |
| `minAge` | Минимальный возраст: `5` |

## 5. Как заполнять города выезда

```typescript
departureCities: [
  { city: 'Краснодар', slug: 'krasnodar', meetingPoint: 'ж/д вокзал', departureTime: '06:00' },
  { city: 'Ростов-на-Дону', slug: 'rostov-na-donu', meetingPoint: 'Главный автовокзал', departureTime: '04:00' },
],
```

**Доступные slug городов:**
- `krasnodar` — Краснодар
- `rostov-na-donu` — Ростов-на-Дону
- `stavropol` — Ставрополь
- `novorossiysk` — Новороссийск
- `sochi` — Сочи

Чтобы добавить новый город выезда → добавьте его также в массив `cities[]` ниже в том же файле.

---

## 6. Как заполнять даты

```typescript
nextDates: [
  { start: '2026-06-14', end: '2026-06-15', price: 5900, seatsLeft: 20 },
  { start: '2026-06-28', end: '2026-06-29', price: 5900, seatsLeft: 15 },
],
```

- Формат даты: `ГГГГ-ММ-ДД`
- `price` — цена именно для этой даты
- `seatsLeft` — количество оставшихся мест
- Опционально: `status: 'confirmed'` / `'few_seats'` / `'sold_out'`

## 8. Как добавлять фото

### Структура папок:

```
public/
  images/
    tours/
      arkhyz-weekend/     ← папка = slug тура
        1.jpg
        2.jpg
        3.jpg
      dombay-5-days/
        1.jpg
        2.jpg
```

### В данных тура:

```typescript
gallery: [
  '/images/tours/arkhyz-weekend/1.jpg',
  '/images/tours/arkhyz-weekend/2.jpg',
  '/images/tours/arkhyz-weekend/3.jpg',
],
```

**Рекомендации по фото:**
- Минимум 3 фото, рекомендуется 4-6
- Горизонтальная ориентация (16:9 или 16:10)
- Размер: 1200-1920px по ширине
- Формат: JPG или WebP
- Первое фото — обложка (будет на карточке в каталоге)

## 9. Доступные бейджи

| Код | На сайте | Когда использовать |
|-----|---------|-------------------|
| `'hit'` | Хит | Самые популярные туры |
| `'new'` | Новинка | Недавно добавленные |
| `'hot'` | Горящий | Скоро отправление / мало мест |
| `'kids'` | С детьми | Подходит для поездок с детьми |
| `'family'` | Семьям | Семейный формат |
| `'weekend'` | Выходные | Тур на 2-3 дня |
| `'sea'` | Море | Морское направление |
| `'mountains'` | Горы | Горное направление |
| `'city'` | Город | Экскурсионный / городской |
| `'train'` | Поезд | Железнодорожный маршрут |
| `'bus'` | Автобус | Автобусный маршрут |

Рекомендуется 2-3 бейджа на тур. Не больше 4.

## 10. Как скрыть тур без удаления

Установите `isPublished: false`:

```typescript
{
  id: '13',
  slug: 'arkhyz-weekend',
  // ... все поля ...
  isPublished: false,  // ← тур скрыт, но данные сохранены
},
```

Тур исчезнет из каталога, главной и страниц городов. Страница тура тоже не будет доступна.

Чтобы вернуть — поменяйте на `true`.

## 11. SEO: title и description

**seoTitle** — заголовок страницы для Google. Суффикс `| KRASKI.TRAVEL` добавляется автоматически.

Примеры хороших seoTitle:
- `'Тур в Архыз на выходные из Краснодара — 2 дня'`
- `'Поездка в Дагестан на 5 дней — горы и экскурсии'`
- `'Семейный тур в Крым из Ростова-на-Дону'`

**seoDescription** — описание для Google. До 160 символов. Должно содержать: куда, откуда, сколько дней, цену.

Пример:
`'Горный отдых в Архызе на выходные: 2 дня, проживание в отеле, экскурсии. Выезд из Краснодара. От 5 900 ₽.'`

## 12. Как проверить, что тур появился

1. Убедитесь, что `npm run dev` запущен
2. Откройте `http://localhost:3000/tours` — тур должен быть в каталоге
3. Откройте `http://localhost:3000/tours/<slug>` — страница тура
4. Откройте `http://localhost:3000/from/<город>` — тур в списке города

## 14. Порядок наполнения сайта

1. **Контакты** → `lib/config.ts`
2. **Первые 3-5 туров** → `data/mock-tours.ts` (заменить mock-туры)
3. **Фото** → `public/images/tours/<slug>/`
4. **SEO** → заполнить `seoTitle` и `seoDescription` для каждого тура
5. **Отзывы** → массив `reviews[]` в `data/mock-tours.ts`
6. **Города и направления** → обновить `cities[]` и `destinations[]`

## 15. Как добавить новый город выезда

Добавьте в массив `cities[]` в `data/mock-tours.ts`:

```typescript
{
  name: 'Волгоград',
  slug: 'volgograd',
  description: 'Описание для SEO-страницы города.',
},
```

Затем используйте этот slug в `departureCities` туров.

---

## 16. Как добавить новое направление

Добавьте в массив `destinations[]` в `data/mock-tours.ts`:

```typescript
{
  name: 'Кисловодск',
  slug: 'kislovodsk',
  region: 'Кавказ',
  description: 'Описание направления.',
  image: '/images/destinations/kislovodsk.jpg',
},
```

Значение `destination` в турах должно совпадать с `name` направления.
